# Kley 🚀

[![Статус сборки](https://github.com/sumbad/kley/actions/workflows/rust.yml/badge.svg)](https://github.com/sumbad/kley/actions/workflows/rust.yml)

**Быстрый и надежный локальный менеджер пакетов для npm (JS/TS), написанный на Rust.**

[English Version (Английская версия)](README.md)

Kley — это утилита командной строки, которая упрощает локальную разработку npm-пакетов. Она предоставляет надежную альтернативу `npm link`, управляя локальным хранилищем пакетов. Вы можете "публиковать" пакеты в центральный кэш на вашем компьютере и "добавлять" их в свои проекты путем прямого копирования файлов, что позволяет избежать распространенных проблем с символическими ссылками.

## Ключевые особенности

- **Очень быстрый**: Написан на Rust для максимальной производительности.
- **Надежный**: Избегает проблем с символическими ссылками, копируя файлы напрямую.
- **Простой API**: Всего две команды для начала работы: `publish` и `add`.
- **Кроссплатформенный**: Работает на macOS, Linux и Windows.

## Установка

_Скоро... (Готовые бинарные файлы будут доступны в разделе Releases)_

На данный момент вы можете собрать проект из исходного кода:
```bash
git clone https://github.com/sumbad/kley.git
cd kley
cargo build --release
# Бинарный файл будет находиться в ./target/release/kley
```

## Как это работает

### 1. `kley publish`

Выполните эту команду в директории пакета, которым вы хотите поделиться локально. Kley:
1.  Прочитает ваш `package.json`.
2.  Скопирует все необходимые файлы в центральное хранилище по пути `~/.kley/packages/<имя-вашего-пакета>`.
3.  Перезапишет любую предыдущую версию этого пакета в хранилище.

Пример:
```bash
# В проекте вашей библиотеки (/path/to/my-lib)
kley publish
```

### 2. `kley add <имя-пакета>`

Выполните эту команду в проекте, где вы хотите использовать ваш локальный пакет. Kley:
1.  Найдет пакет в центральном хранилище.
2.  Скопирует файлы пакета в директорию `./.kley/<имя-вашего-пакета>` внутри вашего проекта.
3.  Предоставит вам инструкцию для добавления локальной зависимости в ваш `package.json`.

Пример:
```bash
# В вашем основном приложении (/path/to/my-app)
kley add my-lib

# Затем добавьте следующее в ваш my-app/package.json:
# "dependencies": {
#   "my-lib": "file:.kley/my-lib"
# }
```
Теперь вы можете выполнить `npm install` и импортировать вашу библиотеку как обычно!
