# f-4: Implement watch command

- **Milestone:** M2

## Description
Implement a `kley watch` command that monitors the current package directory for file changes and automatically runs `publish --push` whenever changes are detected. This creates a seamless hot-reload development loop.

## Acceptance Criteria
1. `kley watch` starts a long-running process that monitors the current directory.
2. File changes trigger an automatic `publish --push`.
3. `.npmignore` and `.kleyignore` are respected for filtering.
4. The process can be stopped with Ctrl+C gracefully.
5. Debouncing is applied to avoid rapid successive publishes.
