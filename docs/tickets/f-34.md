# f-34: Batch PM Installation in `install_all`

- **Milestone:** M2
- **Complexity:** Medium
- **Depends On:** #016 (install command), #033 (--no-save flag)

## 1. Problem Statement

`kley install` (no args) restores all packages from `kley.lock` by calling the package manager in a loop — one process per package. With N packages, N PM processes are spawned sequentially, which is slow and does not leverage the PM's ability to resolve and install dependencies in parallel.

```bash
# Current behavior with 5 packages:
npm install .kley/pkg-a --ignore-scripts   # process 1
npm install .kley/pkg-b --ignore-scripts   # process 2
npm install .kley/pkg-c --ignore-scripts   # process 3
# ...
```

## 2. Proposed Solution

Collect all package paths from `kley.lock` and pass them to a single PM invocation.

```bash
# Desired behavior:
npm install .kley/pkg-a .kley/pkg-b .kley/pkg-c --ignore-scripts
```

### Core Logic

Since `kley install` currently determines `dev`/prod status per package from `package.json`, packages with different statuses cannot be combined into one command. The solution is to split packages into two batches and run at most two PM commands:

```rust
// Split packages into prod and dev lists
let (prod_paths, dev_paths): (Vec<_>, Vec<_>) = lockfile.packages
    .iter()
    .partition(|(name, _)| dev_dependencies.get(*name).is_none());

// At most two PM invocations
if !prod_paths.is_empty() {
    pm_batch_command(&prod_paths, &package, /*dev=*/false, no_save)?;
}
if !dev_paths.is_empty() {
    pm_batch_command(&dev_paths, &package, /*dev=*/true, no_save)?;
}
```

### Interaction with `--no-save`

When `--no-save` is passed, both batches forward the flag to the PM as usual (`--no-save` for npm, `--save=false` for pnpm, no flag for Yarn v1).

### Fast-Path Consideration

The current `install_package` fast-path (Cases A and C) skips the PM when dependencies are unchanged. When introducing batching, packages that qualify for the fast-path should be excluded from the batch before it is assembled, so the PM is only invoked for packages that actually need it.

### Package Manager Support

All three supported PMs accept multiple package paths in a single command:

| PM | Batch command shape |
|----|---------------------|
| npm | `npm install <path-1> <path-2> ... --ignore-scripts` |
| pnpm | `pnpm add <path-1> <path-2> ... --ignore-scripts` |
| Yarn v1 | `yarn add <path-1> <path-2> ... --ignore-scripts` |

## 3. Acceptance Criteria

- `kley install` (no args) spawns at most 2 PM processes regardless of the number of packages in `kley.lock` (one for prod dependencies, one for dev dependencies).
- `--no-save` behavior is preserved in batch mode.
- All existing `install_all` tests pass without modification.
- A new test verifies that with N packages, `pm.log` contains at most 2 lines.
