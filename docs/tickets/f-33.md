# f-33: Add `--no-save` Flag to `install` Command

- **Milestone:** M1
- **Complexity:** Low
- **Depends On:** #016 (install command)

## 1. Problem Statement

When a developer wants to temporarily test a local package in a project, `kley install <pkg>` modifies `package.json` by adding a `file:.kley/<pkg>` dependency. This creates noise in version control — the developer must remember to revert `package.json` before pushing.

The alternative, `kley link`, avoids `package.json` modification but uses a symlink to live source code, which is a different workflow (no PM, no dependency resolution, symlink breaks on `npm install`).

There is no middle ground: install from the store with full PM-driven dependency resolution, but without touching `package.json`.

## 2. Proposed Solution

Add a `--no-save` flag to `kley install` that delegates the `--no-save` behavior to the native package manager. The package is installed into `node_modules/` and tracked in `kley.lock`, but `package.json` remains unchanged.

### Core Logic

```rust
// In main.rs CLI definition
Install {
    name: Option<String>,
    #[arg(long, short = 'D')]
    dev: bool,
    #[arg(long)]
    no_save: bool,   // ← NEW
},
```

```rust
// In install_package() — pass --no-save to PM
let (cmd_name, cmd_args): (&str, Vec<&str>) = match package.manager_type {
    PackageManagerType::Npm => (
        &npm_command,
        vec![
            Some("install"),
            Some(pkg_kley_path_str),
            Some("--ignore-scripts"),
            dev.then_some("--save-dev"),
            no_save.then_some("--no-save"),    // ← NEW
        ]
        .into_iter()
        .flatten()
        .collect(),
    ),
    PackageManagerType::Pnpm => (
        &pnpm_command,
        vec![
            Some("add"),
            Some(pkg_kley_path_str),
            Some("--ignore-scripts"),
            dev.then_some("-D"),
            no_save.then_some("--save=false"),  // ← NEW
        ]
        .into_iter()
        .flatten()
        .collect(),
    ),
    PackageManagerType::Yarn => (
        &yarn_command,
        vec![
            Some("add"),
            Some(pkg_kley_path_str),
            Some("--ignore-scripts"),
            dev.then_some("--dev"),
            // Yarn v1 has no --no-save equivalent.
            // The package.json modification will happen regardless.
            // This is documented as a known limitation.
        ]
        .into_iter()
        .flatten()
        .collect(),
    ),
};
```

### What `--no-save` Changes

| Aspect | `kley install <pkg>` | `kley install <pkg> --no-save` |
|--------|----------------------|-------------------------------|
| Copies to `.kley/<pkg>/` | ✅ | ✅ |
| `kley.lock` updated | ✅ | ✅ |
| `package.json` modified | ✅ `file:.kley/<pkg>` | ❌ |
| PM runs | ✅ | ✅ (with `--no-save`) |
| Package in `node_modules/` | ✅ | ✅ |
| Survives `npm install` | ✅ | ❌ (not in `package.json`) |
| `connection` in lockfile | `"install"` | `"install"` |

### Why No `saved` Field in Lockfile

The `kley remove` command already checks whether a `file:.kley/<pkg>` entry exists in `package.json` before attempting to remove it. If `--no-save` was used, the entry won't be there, and the remove logic skips the `package.json` modification step naturally — no additional field needed.

```rust
// In remove.rs — existing logic already handles this
if let Some(deps_obj) = obj.get_mut(*key).and_then(|d| d.as_object_mut())
    && let Some(dep) = deps_obj.get(package_name)
    && *dep == dep_value   // "file:.kley/<pkg>" — won't match if --no-save was used
    && deps_obj.remove(package_name).is_some()
{
    // Only reaches here if the entry exists in package.json
}
```

### Yarn v1 Limitation

Yarn v1 does not support a `--no-save` flag — it always modifies `package.json`. This is a known limitation. Users on Yarn v1 who need this behavior should use `kley link` instead.

## 3. Usage Examples

```bash
# Test a local package without modifying package.json
kley install my-lib --no-save

# Same with dev dependency
kley install my-lib --no-save -D

# Clean up — just remove, package.json untouched
kley remove my-lib
```

## 4. Acceptance Criteria

- `kley install <pkg> --no-save` installs the package without modifying `package.json`.
- With npm: passes `--no-save` flag to `npm install`.
- With pnpm: passes `--save=false` flag to `pnpm add`.
- With Yarn v1: `package.json` is still modified (documented limitation).
- `kley.lock` is updated normally regardless of `--no-save`.
- `kley remove <pkg>` works correctly after `--no-save` — removes `.kley/<pkg>/`, lockfile entry, and registry entry without attempting to modify `package.json`.
- `kley install` (no args) re-installs all packages from `kley.lock`, including those originally installed with `--no-save`.
