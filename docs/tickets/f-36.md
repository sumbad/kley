# f-36: Implement Trusted Publishing (OIDC) for npm and crates.io

- **Milestone:** M2
- **Complexity:** Medium
- **Depends On:** N/A

## 1. Problem Statement
Currently, the CI/CD pipeline relies on long-lived `NPM_TOKEN` and `CRATES_TOKEN` stored as GitHub Actions Secrets. This presents a security risk, as these tokens could be compromised. Furthermore, managing token rotation is an administrative burden, and these tokens can expire or be revoked, causing CI failures.

## 2. Proposed Solution
Implement **Trusted Publishing** (OIDC) for both npm and crates.io. This removes the need for long-lived secrets by allowing GitHub Actions to request short-lived, scoped tokens directly from the registries.

### Core Changes

#### 2.1. npm Trusted Publishing
- Configure the npm registry to trust the GitHub repository as a publisher.
- Update `package.json` to include `"publishConfig": { "provenance": true }`.
- Update `release.yml` to:
    - Add `permissions: id-token: write`
    - Remove `NPM_TOKEN` usage in the `publish-npm` job.
    - Add `--provenance` flag to `npm publish`.

#### 2.2. crates.io Trusted Publishing
- Add the GitHub repository as a Trusted Publisher on crates.io.
- Update `release.yml` to:
    - Remove `CRATES_TOKEN` usage in the `publish-crates` job.
    - Configure `cargo publish` to use OIDC.

## 3. Acceptance Criteria
- `NPM_TOKEN` and `CRATES_TOKEN` are no longer required in GitHub Secrets.
- GitHub Actions workflow successfully publishes to npm with provenance.
- GitHub Actions workflow successfully publishes to crates.io.
- All existing CI/CD functionality (pre-flight checks, atomic releases) remains intact.
